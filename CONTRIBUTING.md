# Contributing Guide

Thanks for contributing to this showcase-first Unity portfolio repository.

## Contribution Goals
- Keep showcase pages clear, educational, and easy to navigate.
- Keep external source/demo links accurate.
- Improve metadata quality so future entries stay consistent.

## Before You Open A PR
1. Follow the repository format in [docs/PROJECT_FORMAT.md](docs/PROJECT_FORMAT.md).
2. Keep changes focused and minimal.
3. Add or update documentation near your change.
4. Verify all added links are reachable.

## Pull Request Checklist
- I updated `docs/data/games.json` so the sample can be listed in the showcase metadata.
- I updated sample page content in `docs/` if needed.
- I validated `sourceRepo` and `liveDemo` links.
- I updated root [README.md](README.md) sample index if needed.
- I avoided committing generated binaries and large unnecessary files.

## Scope
- This repository is showcase-only.
- Unity project source code should be maintained in external repositories.
- PRs here should focus on docs, metadata, and presentation quality.

## Commit Message Suggestion
Use concise, purpose-first messages, for example:
- `docs(showcase): add Share003_TowerDefense entry`
- `fix(links): update Share002 source repository URL`
- `chore(metadata): refine learning outcomes for Share001`

## Questions
If something is unclear, open an issue with:
- Expected behavior
- Current behavior
- Unity version
- Steps to reproduce
